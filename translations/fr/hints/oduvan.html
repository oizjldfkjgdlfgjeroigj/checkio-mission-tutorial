Avec grand plaisir. C'est à ça que servent les astuces, les conseils, dans CheckiO

Imagine que tu as besoin de multipliser deux nombres et d'afficher le résultat (avec 'print' sur la sortie), de sorte à pouvoir le voir sur la console. Par exemple, 5 and 6.

print(5*6)

Sortie:

30

Maintenant, au lieu de 5 et 6, nous pouvons utiliser des variables. Pourquoi ? Tu vas voir.

a = 5
b = 6
print(a*b)

Sortie:

30

Tu peux modifier la variable a, exécuter à nouveau le script, et voir un résultat différent

a = 6
b = 6
print(a*b)

Sortie:

36

Ou bien tu peux dupliquer ton code, changer les variables et voir le résultat de deux multiplications

a = 5
b = 6
print(a*b)


a = 6
b = 6
print(a*b)

Sortie:

30
36

Maintenant je peux te montrer pourquoi une fonctione simplifiera le code précédent.

def mult_two(a, b):

    return a*b

print(mult_two(5, 6))
print(mult_two(6, 6))

Sortie:

30
36

Comme tu peux le constater, la fonction mult_two contient toute la logique, et tu peux juste changer les paramètre (les arguments) pour voir différents résultats.

Super, merci. Mais tu pourrais m'expliquer la structure de la fonction plus en détails pour que je puisse l'utiliser dans les autres missions CheckiO ?

Bien sûr. Re-voici la fonction :

def mult_two(a, b):

    return a*b

Il faut toujours commencer avec def.

Puis le nom de la fonction mult_two - tu utilisera ce nom plus loin pour appeler la fonction

Deux paramètres (a, b) - ce sont des variables qui prendronts leur valeur au moment où la fonction sera appelée. Par exemple, quand on appelle cette fonction avec mult_two(5, 6), le code à l'intérieur de la fonction utilisera la valeur 5 pour la variable a (le paramètre a) et la valeur 6 pour la variable b (le paramètre b).

Tu peux aussi voir des définitions de fonctions avec des annotations de type.

def mult_two(a: int, b: int) -> int:

    return a*b

Ne te laisse pas impressioner par : int et -> int. Ils aident simplement ceux qui lisent le code le type de paramètre et le type de résultat lors d'un appel à la fonction. Si tu ne les comprends pas encore, tu peux faire simplement comme s'ils n'était pas là.

Ensuite le bloc de la fonction qui commence par quatre espaces et return indique ce que l'appel de fonction doit retourner return a*b.

Dans notre exemple print(mult_two(5, 6)) - le résultat de la multiplication des deux valeurs 5 et 6 sera retourné et ce résultat sera ensuite utilisé comme paramètre de la fonction print.

Si tu as besoin de plusieurs lignes de code pour la fonction - toutes les lignes doivent commencer par quatre espaces. Par exemple, utilisons une troisième variable, nommée c, dans laquelle nous placerons le résultat de la multiplication puis nous retournerons le résultat

def mult_two(a, b):

    c = a*b
    
    return c

D'accord, merci, maintenant c'est un peu plus clair. Mais une fois que j'ai codé ma fonction, que dois-je faire ensuite ?

En haut de l'éditeur de code, tu as deux boutons “Run Code” (exécuter le code) et “Check Solution” (vérifier la solution).

Quand tu cliques sur “Run Code” (exécuter le code) - le code présent dans l'éditeur de code est exécuté par l'ordinateur. Et avec “Check Solution” (vérifier la solution) - l'ordinateur va vérifier ta fonction et afficher “congratulations” (félicitations !) si son code fonctionne correctement.

Si quelque chose ne se passe pas comme attendu - le panneau indiquera que l'appel de la fonction pour lequel la valeur retournée est incorrecte

ou bien lorsqu'il y a une erreur dans un assert
